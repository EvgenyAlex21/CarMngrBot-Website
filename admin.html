Управляйте подписками пользователей, контролируйте функции бота, ведите статистику использования, модерируйте контент, работайте с файлами и резервными копиями. Панель включает продвинутые инструменты для управления пользователями, настройки системы безопасности, мониторинга производительности и обеспечения стабильной работы всех сервисов бота в режиме 24/7.

СИСТЕМНОЕ

Вход в админку

Модуль отвечает за авторизацию администраторов в Telegram-боте, управление сессиями, изменение учетных данных и доступ к админ-панели с поддержкой быстрого входа и проверки прав доступа

Основные функции авторизации:

  • Авторизация: Вход через логин и пароль (основной или индивидуальный для администратора)
  • Быстрый вход: Поддержка быстрого входа для активных сессий (в течение 5 минут после выхода)
  • Проверка статуса: Проверка статуса блокировки и удаленных администраторов
  • Управление сессиями: Хранение активных сессий в JSON-файле с автоматическим удалением через 5 минут

Процесс входа:

  • Запуск через команду /admin в чат с ботом
  • Выбор способа входа: быстрый вход или ввод логина/пароля
  • Автоматическое хэширование паролей с использованием SHA-256
  • Доступ к админ-панели с полным набором функций управления
СИСТЕМНОЕ

Управление системой

Комплексное управление подписками, магазином, баллами, обменами, скидками, подарками и статистикой системы с полным контролем всех операций

Основные модули системы:

  • Управление подписками: Добавление, просмотр и удаление подписок пользователей
  • Управление магазином: Начисление покупок, просмотр истории и удаление операций
  • Управление баллами: Начисление, списание, просмотр баланса и истории операций
  • Управление обменами: Обмен баллов на время, скидки или функции
  • Управление скидками: Создание промокодов и назначение персональных скидок
  • Управление подарками: Организация передачи ресурсов между пользователями
  • Статистика системы: Комплексная аналитика всех модулей с экспортом данных

Возможности управления:

  • Централизованный доступ ко всем системным функциям
  • Автоматические уведомления пользователей об операциях
  • Проверка прав доступа для каждого модуля
  • Интегрированная система отчетности и аналитики
СИСТЕМНОЕ

Админ

Комплексное управление администраторами: смена данных входа, добавление и удаление администраторов, настройка прав доступа с гибкой системой безопасности

Управление учетными записями:

  • Смена данных входа: Изменение логина и пароля текущего администратора
  • Добавить админа: Назначение новых администраторов с базовыми правами
  • Удалить админа: Отзыв административных прав с переносом в архив
  • Права доступа: Гибкое управление разрешениями для каждого модуля

Система безопасности:

  • Проверка прав доступа перед выполнением операций
  • Защита от удаления корневого администратора и самого себя
  • Хэширование паролей с использованием SHA-256
  • Автоматическое удаление конфиденциальных сообщений
  • Логирование всех административных действий

Права доступа включают:

  • Управление системой (подписки, магазин, баллы, обмены, скидки, подарки)
  • Администрирование (добавление/удаление админов, права доступа)
  • Модерация (бан пользователей, управление функциями)
  • Аналитика (статистика, файлы, резервные копии)
  • Коммуникации (общение, реклама, редакция новостей)
СИСТЕМНОЕ

Бан

Позволяет администратору блокировать или разблокировать пользователей, изменяя их статус в базе данных, а также полное удаление информации о пользователе из всех файлов в системе

Основные возможности:

  • Отображает список всех пользователей с их статусом (заблокирован/разблокирован)
  • Предлагает выбрать действие: заблокировать, разблокировать или удалить данные
  • Защита от блокировки корневых администраторов и самого себя
  • Автоматические уведомления о изменении статуса
СИСТЕМНОЕ

Функции

Позволяет администратору активировать или деактивировать определенные функции. Функции можно отключить на заданный срок с автоматическим включением по истечении времени

Основные возможности:

  • Предлагает включить или выключить функции через меню
  • Планирование автоматического включения функций
  • Отслеживание времени деактивации функций
СИСТЕМНОЕ

Статистика

Показывает статистику пользователей (онлайн, за день/неделю/месяц/год), собирает данные об использовании функций и экспортирует их в Excel, отображает лог ошибок, версию бота и аптайм

Основные возможности:

  • Предлагает подменю: пользователи, использование функций, список ошибок, версия и аптайм
  • Экспорт статистики в Excel-файлы
  • Мониторинг системных показателей в реальном времени
СИСТЕМНОЕ

Файлы

Поиск файлов по расширению (EXT), директории (DIR) или ID пользователя, добавление новых файлов в выбранную директорию, замена существующих файлов, удаление файлов из выбранной директории

Основные возможности:

  • Предлагает меню с опциями: поиск по EXT, DIR, ID, добавление, замена и удаление файлов
  • Поддержка навигации по файловой системе
  • Временное хранилище для безопасной обработки файлов
СИСТЕМНОЕ

Резервная копия

Создание полной и инкрементной резервной копии данных, восстановление данных из последней резервной копии, автоматическое создание копий по расписанию с очисткой старых файлов, мониторинг использования диска

Основные возможности:

  • Предлагает меню: создать копию или восстановить данные
  • Автоматическое расписание резервного копирования
  • Проверка целостности архивов и мониторинг дискового пространства
СИСТЕМНОЕ

Общение

Модуль предоставляет администраторам различные способы коммуникации с пользователями: инициирование чатов, управление запросами, отправка оповещений и просмотр диалогов

Основные возможности:

  • Чат: Инициирование чата с пользователем по номеру, ID или username
  • Запросы: Управление запросами на чат от пользователей
  • Оповещения: Отправка сообщений всем или отдельным пользователям, включая планирование по времени
  • Диалоги: Просмотр и удаление истории переписки с пользователями
СИСТЕМНОЕ

Реклама

Модуль позволяет администраторам управлять рекламными запросами: принимать, отклонять, отправлять рекламу (всем или по расписанию) и удалять её

Основные возможности:

  • Предоставляет выбор: "Запросы на рекламу", "Удалить рекламу", "В меню админ-панели"
  • Поддержка текста и мультимедиа (фото, видео, документы и т.д.)
  • Автоматическая фоновая проверка сроков действия рекламы
СИСТЕМНОЕ

Редакция

Модуль позволяет администраторам публиковать, редактировать, просматривать и удалять новости. Поддерживаются заголовки, текст, подписи и мультимедиа

Основные возможности:

  • Предоставляет выбор: "Опубликовать новость", "Отредактировать новость", "Посмотреть новость", "Удалить новость"
  • Поддержка мультимедиа файлов (до 10 файлов на новость)
  • Автоматическая переиндексация при удалении новостей
СИСТЕМНОЕ

Экстренная остановка

Модуль позволяет администраторам полностью останавливать бота с обязательным подтверждением и 5-секундным таймером безопасности

Ключевые функции системы:

  • emergency_stop: Инициирует запрос подтверждения остановки
  • confirm_emergency_stop: Обрабатывает подтверждение или отмену решения
  • stop_bot_after_delay: Запускает 5-секундный таймер для остановки
  • stop_bot: Останавливает polling и завершает процесс (os._exit(0))
СИСТЕМНОЕ

Выход

Выход из админ-панели с завершением сессии администратора и возможностью быстрого повторного входа

Процесс завершения сессии:

  • Безопасное завершение текущей административной сессии
  • Очистка временных данных и состояний
  • Сохранение важных изменений перед выходом
  • Логирование времени завершения сессии для аудита

Быстрый повторный доступ:

  • В течение 5 минут после выхода возможен быстрый повторный вход
  • Система запоминает последнюю активную сессию
  • Упрощенная процедура авторизации в период "быстрого доступа"
  • Автоматическое восстановление предыдущих настроек интерфейса

Безопасность:

  • После истечения 5-минутного периода требуется полная авторизация
  • При подозрительной активности быстрый доступ блокируется
  • Автоматический выход при длительной неактивности

Управление системой

СИСТЕМНОЕ

Управление подписками

Назначение стандартного или индивидуального плана подписки пользователю, отображение списка подписок с деталями (тип, даты, стоимость, статус), удаление активных подписок с уведомлением пользователя

Основные возможности:

  • Назначение стандартных и индивидуальных планов подписки
  • Детальный просмотр всех подписок пользователя с аналитикой
  • Управление активными подписками с возможностью удаления
  • Автоматические уведомления и корректировка статусов доступа
СИСТЕМНОЕ

Управление магазином

Добавление баллов или времени подписки пользователю, просмотр истории покупок с итоговой оценкой, удаление покупок с корректировкой баллов и статистики

Основные возможности:

  • Начисление покупки: Добавление баллов или времени подписки с автоматическими уведомлениями
  • Просмотр покупок: Просмотр истории покупок пользователя с полной детализацией и итоговой оценкой
  • Удаление покупок: Удаление покупок с автоматической корректировкой баллов, времени и статистики

Типы операций в магазине:

  • Покупка баллов: Максимум 3000 баллов в месяц на пользователя
  • Покупка времени подписки: Максимум 365 дней в месяц на пользователя
  • Автоматическое ведение истории всех операций
  • Корректировка балансов при удалении покупок

Система уведомлений:

  • Автоматические уведомления администратору и пользователю при каждой операции
  • Детальная информация о типе, количестве и стоимости покупки
  • Мгновенное обновление балансов и подписок
СИСТЕМНОЕ

Управление баллами

Позволяет администратору начислять баллы выбранным пользователям, отображает статистику баллов и историю операций, уменьшает количество баллов у пользователя с учетом доступного баланса

Операции с баллами:

  • Начисление баллов: Добавление баллов пользователю с указанием причины и автоматическими уведомлениями

Экстренная остановка

СИСТЕМНОЕ

Подтвердить остановку

Критическая остановка бота с подтверждением и таймером безопасности

Процесс экстренной остановки:

  • Система запрашивает подтверждение остановки бота
  • При подтверждении запускается 5-секундный таймер (stop_bot_after_delay)
  • Бот уведомляет всех администраторов о предстоящей остановке
  • Автоматически останавливается polling и завершается процесс (os._exit(0))
  • Все активные операции прерываются немедленно

Технические детали:

  • emergency_stop: Инициирует запрос подтверждения
  • confirm_emergency_stop: Обрабатывает решение администратора
  • stop_bot: Выполняет финальную остановку системы
СИСТЕМНОЕ

Отмена остановки

Отмена процедуры экстренной остановки с возвратом к нормальной работе

Процесс отмены:

  • Прерывает процедуру экстренной остановки
  • Отменяет запущенный таймер остановки
  • Возвращает администратора в главное меню админ-панели
  • Бот продолжает нормальную работу без прерываний
  • Уведомляет о успешной отмене операции

Условия отмены:

  • Отмена возможна только до истечения 5-секундного таймера
  • После подтверждения отмены все системы возвращаются в рабочее состояние
  • История попытки остановки сохраняется в логах для анализа

Для выбора способа входа в админ-панель:

    • Выбрать "Быстрый вход" (если учетные данные не изменены)
    • Выбрать "Ввести логин и пароль заново"
    • Выбрать "В главное меню" для возврата
  2. Если сессия неактивна или новый администратор:
    • Ввести логин (для новых администраторов потребуется смена данных)

Процесс ввода учетных данных:

  3. Сообщения с логином и паролем автоматически удаляются для безопасности
  4. Система выполняет хэширование SHA-256 для проверки данных

Примечание: Все конфиденциальные данные немедленно удаляются из чата после обработки

  2. Ввести новый уникальный логин (минимум 3 символа)
  3. Установить надежный пароль (минимум 8 символов с заглавными буквами, цифрами и спецсимволами)
  4. После смены данных получить доступ к админ-панели

Примечание: Смена данных входа обязательна для всех новых администраторов

Примечание: Экстренная остановка полностью завершает работу бота. Используйте только при критических сбоях системы или угрозе безопасности данных.

СИСТЕМНОЕ

12.2. Отмена остановки

Отмена экстренной остановки:

  1. В течение 5 секунд после подтверждения возможна отмена
  2. Нажмите кнопку "Отменить остановку"
  3. Система отменит процедуру остановки
  4. Бот продолжит нормальную работу

Примечание: Окно для отмены составляет всего 5 секунд. После истечения этого времени остановка будет выполнена автоматически и отменить её будет невозможно.

СИСТЕМНОЕ

13. Выход

Для безопасного выхода из админ-панели:

  1. Завершите все активные операции
  2. Сохраните важные изменения
  3. Нажмите кнопку "Выход"
  4. Сессия будет завершена автоматически

Примечание: Всегда выходите из админ-панели после завершения работы для обеспечения безопасности

СИСТЕМНОЕ

14. Вспомогательные функции

СИСТЕМНОЕ

14.1. Состояние функций

Проверка доступности функций:

  1. Система автоматически анализирует состояние каждой функции бота
  2. При попытке использовать функцию, её статус проверяется:
    • Если функция доступна — запрос обрабатывается
    • Если функция недоступна — выводится уведомление

Примечание: Функции могут быть временно недоступны из-за технического обслуживания или обновления. При этом пользователь получает информативное сообщение о причине недоступности

СИСТЕМНОЕ

14.2. Статистика функций

Учёт использования функций:

  1. Система фиксирует каждое обращение к функциям бота
  2. Статистические данные собираются для анализа:
    • Популярности конкретных функций
    • Частоты использования различных сервисов
    • Оптимизации работоспособности бота

Примечание: Все данные собираются в обезличенной форме и используются исключительно для улучшения работы бота. Персональная информация пользователей при этом не раскрывается!

СИСТЕМНОЕ

14.3. Блокировка пользователей

Проверка статуса пользователя:

  1. При каждом обращении к боту система проверяет статус пользователя
  2. Если администратор заблокировал доступ пользователя:
    • Запросы не обрабатываются
    • Отображается сообщение о блокировке

Примечание: Блокировка применяется в случаях нарушения правил использования бота, массовой рассылки запросов или неправомерных действий. Для разблокировки необходимо связаться с администратором через указанные контакты

СИСТЕМНОЕ

14.4. Обновление активности

Отслеживание активности пользователя:

  1. При любом взаимодействии с ботом система регистрирует активность
  2. Обновляются следующие параметры:
    • Дата и время последнего действия
    • Тип выполненной операции
    • Статус использования бота

Примечание: Данная функция необходима для корректной работы бота с вашей активностью

СИСТЕМНОЕ

14.5. Уведомления о неактивности

Для управления неактивностью:

  1. При отсутствии активности в течение 30 дней вы получите уведомление
  2. Если активность не возобновится, данные будут удалены через 60 дней без возможности восстановления

Примечание: Для возобновления активности достаточно прописать команду /start или использовать любую из функций

СИСТЕМНОЕ

14.6. Состояние чата

Контроль состояния диалога:

  1. Система отслеживает текущее состояние диалога с пользователем
  2. При обработке запросов:
    • Если предыдущий запрос обрабатывается — новые запросы ставятся в очередь
    • Если возникла ошибка — отображается соответствующее уведомление
    • При нормальной работе — обеспечивается последовательный обмен сообщениями

Примечание: Данная функция предотвращает действия, когда происходит чат между администратором и пользователем

СИСТЕМНОЕ

14.7. Логирование ошибок

Регистрация событий и ошибок:

  1. Система автоматически фиксирует все действия:
    • Запросы пользователя (без персональных данных)
    • Ответы бота
    • Технические ошибки и сбои
  2. Записи журнала используются для:
    • Выявления и исправления ошибок
    • Оптимизации работы бота
    • Обеспечения стабильности сервиса

Примечание: Логи содержат только технические данные и не включают личную информацию пользователя! Срок хранения логов составляет 30 дней, после чего они автоматически удаляются

СИСТЕМНОЕ

14.8. Блокировка бота

Обработка случаев блокировки бота:

  1. Система определяет, если пользователь заблокировал бота в Telegram
  2. При обнаружении блокировки:
    • Прекращается отправка уведомлений
    • Данные пользователя сохраняются в течение 30 дней
    • По истечении срока данные удаляются без возможности восстановления

Примечание: При разблокировке бота в течение 30-дневного периода все данные и настройки сохраняются. Для возобновления работы достаточно отправить любое сообщение или команду /start

СИСТЕМНОЕ

14.9. Проверка функций

Проверка доступа к функциям:

  1. При каждом запросе система проверяет статус подписки пользователя
  2. Распределение доступа:
    • Бесплатные функции доступны всем пользователям
    • Платные функции доступны только при активной подписке
    • При запросе платной функции без подписки отображается предложение оформить подписку

Примечание: Система автоматически отслеживает срок действия подписки и за несколько дней до её окончания отправляет уведомление с предложением продления

СИСТЕМНОЕ

14.10. Подписка канал

Проверка статуса подписки на канал:

  1. При активации бота и регулярно во время использования система проверяет наличие подписки на основной канал бота
  2. Статусы проверки:
    • Если пользователь подписан — предоставляется полный доступ
    • Если подписка отсутствует — отображается сообщение с просьбой подписаться

Примечание: Подписка на канал является обязательным условием использования бота, так как через канал рассылается важная информация об обновлениях и изменениях в работе сервиса

СИСТЕМНОЕ

14.11. Текстовые сообщения

Обработка типов сообщений:

  1. Система анализирует тип входящего сообщения от пользователя
  2. Возможные действия в зависимости от типа сообщения:
    • Текстовое сообщение — обрабатывается стандартным образом
    • Мультимедийные файлы (фото, видео, документы и пр.) — обрабатываются только в специальных режимах (например, в чате с администратором)
    • В других случаях выводится сообщение о необходимости ввода текстовой информации

Примечание: Данное ограничение необходимо для корректной работы бота и предотвращения ошибок в обработке команд. В разделах, где поддерживается прием мультимедийных сообщений, будет указано соответствующее уведомление

СИСТЕМНОЕ

14.12. Капча

Защита от массовых запросов:

  1. При превышении лимита запросов (более 5 за 10 секунд) система активирует капчу
  2. Процесс прохождения капчи:
    • Пользователю предлагается решить простую математическую задачу
    • При правильном ответе доступ восстанавливается
    • При неправильном ответе предлагается новая задача

Примечание: Данный механизм защищает бота от перегрузки и автоматизированных атак. При добросовестном использовании пользователь редко сталкивается с капчей. Если капча появляется слишком часто, рекомендуется делать паузы между запросами
